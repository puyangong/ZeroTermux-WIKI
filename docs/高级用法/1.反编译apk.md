# 反编译apk(BY PYG)

## 1.准备工作

### 1.切换清华源

### 2.安装需要的库

```
pkg install apktool apksigner
```

### 3.文件管理器

mt管理器

## 2.解包apk

```
apktool d xxx.apk
```

将apk解压到当前目录下名为xxx的文件夹中

若你想使用自定义的文件夹

```
apktool d xxx.apk ./xxx2
```

## 3.修改apk

解包后可以进入文件夹修改apk，反编译修改apk是一件十分复杂繁琐的事情，此处建议直接在终端内使用ai实现修改

同时对于解压出的内容，可以直接用mt管理器的搜索功能搜索具体变量或函数，十分方便。

## 4.打包apk

```
apktool b xxx.apk
```

将apk打包到当前目录下xxx/dist路径中

注意，此时打包的apk没有签名，所以需要签名apk。

签名步骤：

```
# 生成签名密钥（仅需一次）
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-alias
```

| 参数                           | 含义                                                         |
| ------------------------------ | ------------------------------------------------------------ |
| `-genkey`                      | **生成密钥对**，同时生成自签名证书                           |
| `-v`                           | **verbose 模式**，显示详细过程信息                           |
| `-keystore my-release-key.jks` | **密钥库文件路径**，`.jks` = Java KeyStore 格式，这是存储密钥和证书的数据库文件 |
| `-keyalg RSA`                  | **密钥算法**，RSA 是 Android 签名的标准算法                  |
| `-keysize 2048`                | **密钥长度**，2048 位是安全与性能的平衡点（Android 最低要求 1024，推荐 2048+） |
| `-validity 10000`              | **有效期天数**，约 27 年，确保密钥在应用生命周期内有效       |
| `-alias my-alias`              | **密钥别名**，密钥库中这条密钥的"名字"，后续签名时需要引用   |

```
# 签名 APK
apksigner sign --ks my-release-key.jks --ks-key-alias my-alias --out signed.apk unsigned.apk
```

| 参数                      | 含义                                                       |
| ------------------------- | ---------------------------------------------------------- |
| `sign`                    | **子命令**：执行签名操作                                   |
| `--ks my-release-key.jks` | **密钥库路径**，指向刚才生成的 `.jks` 文件                 |
| `--ks-key-alias my-alias` | **指定使用哪个密钥**，一个密钥库可存多条密钥，通过别名选择 |
| `--out signed.apk`        | **输出路径**，签名后的 APK 保存位置                        |
| `unsigned.apk`            | **输入文件**，待签名的 APK（通常是构建或修改后的未签名包） |

签名后即可正常安装测试apk。
